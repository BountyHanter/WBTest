# 📊 Работа с WB API: статистика и изображения

## 1. Получение статистики по рекламе

### 📌 Эндпоинт

```
GET https://advert-api.wildberries.ru/adv/v3/fullstats
```

### 📌 Параметры

* `beginDate` — дата начала (YYYY-MM-DD)
* `endDate` — дата конца (YYYY-MM-DD)

---

## 📥 Структура ответа (важное)

### 🔝 Верхний уровень (АГРЕГАЦИЯ)

Используется для быстрой работы:

```
views
clicks
```

👉 Это сумма по всему диапазону дат

---

### 📅 days[]

Разбивка по дням:

```
days[].date
days[].views
days[].clicks
```

---

## ⚠️ Особенности API

* Статистика **накопительная**
* В течение дня значения **растут**
* По дням данные **разделены**
* Возможны **пересчёты WB (редко)**

---

## ✅ Рекомендуемый подход

### Вариант 1 (проще)

Использовать верхний уровень:

```
total_views
total_clicks
```

Хранить:

```
last_total_views
last_total_clicks
```

Считать:

```
delta = current - last
```

---

### Вариант 2 (надёжнее)

Использовать `days[]`

Хранить:

```
last_day
last_day_views
last_day_clicks
```

---

### 🔥 Алгоритм

1. Запросить статистику
2. Найти нужный день
3. Если день тот же:

   ```
   delta = current - last
   ```
4. Если день сменился:

   ```
   delta = today + (yesterday_total - last_yesterday)
   ```

---

## ⚠️ Обработка ошибок

```
if current < last:
    сбросить baseline (пересчёт WB)
```

---

# 🖼 Работа с изображениями

## 📌 Эндпоинт загрузки

```
POST https://content-api.wildberries.ru/content/v3/media/file
```

---

## 📌 Заголовки

```
Authorization: <API_KEY>
X-Nm-Id: <nmId товара>
X-Photo-Number: <номер изображения>
```

---

## 📌 Body (multipart)

```
uploadfile=@/path/to/file.jpg
```

---

## 📌 Пример curl

```
curl -X POST "https://content-api.wildberries.ru/content/v3/media/file" \
  -H "Authorization: <API_KEY>" \
  -H "X-Nm-Id: 123456" \
  -H "X-Photo-Number: 1" \
  -F "uploadfile=@/path/to/image.jpg"
```

---

## ⚠️ Важно

### Номер изображения

* `1` → заменить первую картинку
* `2+` → добавить новую

---

### Ограничения

* до 30 изображений
* минимум 700x900
* максимум 32 МБ

---

# 🚨 Кэширование (КРИТИЧНО)

## ❗ Проблема

После загрузки изображения:

* браузер может показывать старое
* CDN WB кеширует

---

## 📌 Симптомы

* обычный браузер → старая картинка
* инкогнито → новая

---

## 📌 Причина

Кэш:

* CDN (основное)
* cookies
* регион
* user-agent

---

## ⚠️ ВАЖНО ДЛЯ A/B ТЕСТОВ

После смены изображения:

👉 пользователи **не сразу видят новое**

---

## ✅ Что делать

### 1. Задержка после смены

```
1–5 минут
```

---

### 2. Не считать статистику сразу

```
игнорировать первые N минут
```

---

### 3. Проверка

* инкогнито
* разные браузеры

---

# 🔁 Алгоритм A/B теста (упрощённо)

1. Получить статистику
2. Посчитать дельту
3. Обновить данные изображения:

   * views
   * clicks
4. Проверить условия:

   * время круга
   * лимит показов
5. Если условие выполнено:

   * сменить изображение
   * записать время смены
6. Подождать (кэш!)
7. Продолжить сбор статистики

---

# 🚀 Ключевые выводы

* WB API даёт **накопительную статистику**
* Дельта считается **всегда вручную**
* CDN ломает "мгновенные" тесты
* Без задержки после смены — тест будет кривой
* Лучше использовать `days[]`, если нужна надёжность
