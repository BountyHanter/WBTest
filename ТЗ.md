# 📄 ТЗ: Сервис ротации изображений WB (финальная версия)

---

## 1. Общая логика

Сервис управляет ротацией изображений товара в рекламной кампании WB и собирает статистику (показы и клики) для каждого изображения.

Система:

```text
- поочередно показывает изображения
- собирает статистику через baseline
- фиксирует результат по завершению цикла показа изображения
- повторяет процесс, пока изображения не исчерпают лимит
- завершает тест, когда все изображения достигли лимита показов
```

* id

* product_id (nmId)

* name

  * draft

### ⚙️ Настройки теста

* impressions_per_cycle — лимит показов за один цикл изображения
* max_impressions_per_image — общий лимит показов изображения
* time_per_cycle — максимальное время показа изображения (сек)

---

### 🖼 Image

* id

* test_id

* position

* image_url

* status:

  * pending
  * done (достигнут max_impressions_per_image)

---

### 📊 Статистика

* total_views
* total_clicks
* rounds_passed

---

### ⚠️ Временные поля

* baseline_views
* baseline_clicks
* started_at

## 5. Cron-задача

---

### 🔁 Запуск

```text
каждую минуту
```

---

### 🔁 Батчевая обработка

Используется поле:

```text
test.is_processed
```

Алгоритм:

1. выбираются тесты:

```text
status = active AND is_processed = false
```

2. берётся ограниченное количество (например 100–200)

3. обрабатываются

4. после обработки:

```text
is_processed = true
```

5. если не осталось тестов с is_processed = false:

```text
у всех active тестов → is_processed = false
```

## 6. Алгоритм работы теста

---

### 1. Получение статистики

Запрос к WB:

```text
GET /adv/v3/fullstats
```

Получаем накопительные:

* views
* clicks

---

### 2. Расчёт дельты

```text
delta = current - baseline
```

Если:

```text
delta < 0
```

→ логируем и пропускаем тик

---

### 3. Проверка завершения цикла изображения

Цикл завершён если:

```text
delta_views >= impressions_per_cycle
ИЛИ
(now - started_at) >= time_per_cycle
```

---

### 4. При завершении цикла

```text
total_views += delta_views
total_clicks += delta_clicks
rounds_passed += 1

baseline = current
started_at = now
```

---

### 5. Проверка лимита изображения

```text
если total_views >= max_impressions_per_image:
    image.status = done
```

---

### 6. Выбор следующего изображения

Берётся:

```text
изображение с минимальным position
у которого total_views < max_impressions_per_image
```

---

### 7. Переключение изображения

```text
1. выбирается следующая картинка
2. отправляется в WB
3. test.current_image обновляется
```

---

### ⚠️ ВАЖНО: подтверждение переключения

```text
❗ ВОПРОС (открытый):

система не имеет подтверждения,
что WB реально применил изображение

Решение:
- либо считать переключение успешным сразу
- либо добавлять механизм проверки (если появится API)

решается отдельно
```

---

### 8. Завершение теста

```text
если НЕТ изображений с total_views < max_impressions_per_image:
    test.status = finished
    test.current_image = null
```

## 7. Пауза и возобновление

---

### ⏸ Pause

```text
1. запрос к WB
2. считаем delta
3. сохраняем в total
4. status = paused
```

---

### ▶ Resume

```text
1. запрос к WB
2. baseline = current
3. started_at = now
4. status = active
```

## 11. Что важно помнить

```text
- baseline — только для расчёта
- total — только при завершении цикла
- картинка живёт пока не достигнет лимита
- цикл ≠ завершение картинки
```
